# Concetti fondamentali



**Datacenter:** È un centro di elaborazione di dati in un edificio fisico, contiene tanti server.  
All'interno del datacenter i server vengono raggruppati in cluster (es. cluster da 50 server che vengono uniti per farli lavorare insieme come se fossero un unico cervello).  



**Server:** È un PC (ha processore, RAM, scheda madre) che ha scopo e forma completamente diversi da quella dei PC tradizionali.  
- forma: È una piastra piatta e lunga che viene impilata in cassetti chiamati server rack
- non ha schermo ne scheda audio, lo si comanda a distanza con ssh 
- usa una distribuzione linux in versione server come OS


ILo scaling verticale va incontro ai limiti fisici e monetari e per questo nasce lo scaling orizzontale.  
Invece di avere una singola macchina più potente, si creano dei cluster di server (chiamati nodi) e li si fanno lavorare insieme in due modi:

- High Availability: se un server si rompe gli altri continuano a lavorare 
- Load Balancing: divido il lavoro tra tutti i server

Fisicamente la scatola di metallo è sempre un server, il suo scopo specifico dipende dal software che ci installiamo e dal hardware che ci mettiamo dentro

- web server: servono le pagine HTML o le API, hanno bisogno di molta connessione di rete e CPU media 
- database server: conservano i dati. necessitano di dischi (SSD) ultra veloci e tantissima RAM
- compute server (HPC): fanno calcoli matematici estremi, hanno bisogno di processori potentissimi e schede video  


**es: Instsagram**   
Instagram per servire milioni di utenti al giorno usa un cluster di web server e un load balancer.  

- load balancer: quando un utente apre l'app di instagram non si collega ai server veri, si collega al load balancer che ha un unico indirizzo IP pubblico raggiungibile da tutto il mondo   
- web server (i cloni): dietro al load balancer ci sono ad esempio 1000 server web, tutti identici, contengono esattamente lo stesso codice dell'applicazione. Non esiste un main server, sono tutti uguali.  
- la distribuzione: arriva una richiesta "voglio vedere la foto di X"; allora il laod balancer guarda i suoi 1000 server che gestisce e magari vede che il server 42 è il più libero in quel istante, gli gira quindi la richiesta e il server 42 processerà la richiesta!   

I web server sono progettati per essere **Stateless** (senza stato/memoria), sui loro dischi c'è solo il codice dell'applicazione (l'algoritmo) ma non c'è nessuna foto/password.  
Quando il server 42 riceve la richiesta, si gira verso un altro cluster, ossia il **Cluster di Database**, da questo prende la foto che gli serve e la restituisce all'utente.
