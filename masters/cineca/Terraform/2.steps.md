# Terraform steps  


Creare la cartella principale del progetto dalla quale verrà invocato terraform    


creare i seguenti file:  

`versions.tf`: qui definiremo il provider (openstack)  
`variables.tf`: l'insieme di variabili, dove viene definito il tipo della variabile e il valore di default  
`cluster.auto.tfvars`: si definiscono le variabili che verranno usate, con i loro valori!  
`main.tf` : è il codice esecutivo, spartisce i comandi a openstack richiamando i moduli che sono definiti al suo interno.  
    Dentro al main chiamiamo i moduli passandogli le variabili!  

accesso alle variabili: in variables.tf abbiamo 

nota: è buona norma e buon design pattern mettere in questa dir tutte le variabili, anche quelle che useremo dentro i moduli.  
Questo per garantire riusabilità e per avere una singola fonte di verità.  

### moduli:  

Nel modulo definiamo i seguenti file:  

versions.tf (opzionale): serve per dichiarazione di compatibilità, il vero plugin è scaricato dalla root directory, tale version deve coincidere con la specifica dichiarata qui (es. >= 1.26)    

variables.tf: questo file serve come **contratto api**, le variabili dichiarate vanno lasciate vuote. Prenderà le variabili dal file .tfvars esistente nella root directory.  

main.tf 

outputs.tf: serve per ottenere le informazioni create.   
    es: `value = element(concat(openstack_networking_subnet_v2.k8s.*is, [""]),0)`  
    prende l'id della subnet che ha appena creato terraform e lo sputa fuori.  
    Grazie a questo output, nel main.tf della root possiamo fare: `private_subnet_id = module.network.subnet_id`
